# Сессионное задание по дисциплине "Базы данных"

Выполнил: Ярцев Владислав Дмитриевич

Репозиторий с решениями сессионной работы по четырём базам данных из учебного задания: для каждой - схема и данные по условию и запросы-решения к задачам.

## Цель и задачи работы

**Цель** - провести анализ схем баз данных из условия задания и реализовать сложные запросы к ним: соединения (`JOIN`), объединение выборок (`UNION`), группировка и условия к ним (`HAVING`), а также рекурсивные.

**Задачи** - для каждой из четырёх баз (см. таблицу ниже) поднять схему и данные по условию (`create-and-fill.sql`) и написать решения задач на выборку (`tasks.sql`).

По условию задания выделяются следующие базы:

| № | Название по условию | Папка в репозитории |
|---|---------------------|---------------------|
| 1 | **База данных 1. Транспортные средства** | `1-vehicles/` |
| 2 | **База данных 2. Автомобильные гонки** | `2-auto-races/` |
| 3 | **База данных 3. Бронирование отелей** | `3-hotel-reservations/` |
| 4 | **База данных 4. Структура организации** | `4-organization-structure/` |

## Структура репозитория

В каждой папке содержатся два файла:

- **`create-and-fill.sql`** - скрипт, который создаёт все таблицы нужной базы в том виде, как задано по условию, задаёт связи между ними и заполняет таблицы тестовыми строками, чтобы на них можно было прогонять запросы из задач (код получен из условия задания).
- **`tasks.sql`** - решения задач (запросы с комментариями по блокам).

## Схемы данных

### База данных 1. Транспортные средства (`1-vehicles/`)

- **Vehicle** - производитель, модель, тип (`Car` / `Motorcycle` / `Bicycle`), первичный ключ по модели.
- **Car**, **Motorcycle**, **Bicycle** - детали по типу транспорта. Модель ссылается на `Vehicle`.

В запросах: `JOIN` по модели, `UNION ALL` для объединения выборок по разным таблицам, фильтрация и сортировка.

### База данных 2. Автомобильные гонки (`2-auto-races/`)

- **Classes** - класс автомобиля (тип, страна, двери, двигатель, масса).
- **Cars** - экземпляр машины в классе.
- **Races** - гонка и дата.
- **Results** - место машины в гонке (составной ключ "машина + гонка").

В запросах: агрегаты по машинам и классам, подзапросы, `LIMIT`, нерекурсивные CTE (`WITH`).

### База данных 3. Бронирование отелей (`3-hotel-reservations/`)

- **Hotel**, **Room** - отель и номер с типом и ценой.
- **Customer** - клиент.
- **Booking** - бронь номера клиентом на период.

В запросах: `JOIN` цепочкой по бронированиям, `GROUP BY`, `HAVING`, `DATEDIFF`, `GROUP_CONCAT`, подзапрос для категоризации отелей по средней цене номеров, `CASE` / `FIELD` для приоритета категорий.

### База данных 4. Структура организации (`4-organization-structure/`)

- **Departments**, **Roles** - справочники.
- **Employees** - сотрудник, менеджер (`ManagerID` на ту же таблицу), отдел, роль.
- **Projects** - проект привязан к отделу.
- **Tasks** - задача назначена сотруднику и проекту.

В запросах: **рекурсивный CTE** для обхода дерева подчинения, `GROUP_CONCAT` для списков проектов и задач, подсчёт подчинённых и задач.

## Как запустить и проверить

1. Создайте базу данных и выберите её. Имя в MySQL может быть любым латиницей, например отдельно под каждую тему: `transport`, `races`, `hotels`, `organization` - или одна временная `session_db`, если проверяете блоки по очереди.

   ```sql
   CREATE DATABASE session_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
   USE session_db;
   ```

2. Выполните (в оболочке клиента mysql) `create-and-fill.sql` для нужной базы по условию (папка из таблицы выше), например:

   ```text
   source /полный/путь/к/репозиторию/1-vehicles/create-and-fill.sql
   ```

   Либо откройте файл и выполните целиком.

3. Выполните запросы из `tasks.sql` той же папки - по одному или целиком.

4. Перед переходом к очередной базе из задания пересоздайте схему или используйте другую `DATABASE`, чтобы не смешивать таблицы разных блоков (в частности, во второй базе есть таблица `Cars`, не совпадающая по смыслу с `Car` из первой).

Проверка: после `create-and-fill.sql` не должно быть ошибок DDL/DML. Каждый запрос из `tasks.sql` должен выполняться без ошибки.
